# Развитие песочницы: Подсистема боковой панели (Sidebar)

Для реализации плагинов со сложным и широким интерфейсом (включая будущий плагин "Стратегическая коуч-сессия") приложению требуется система левой боковой панели и разделения экрана.

## Задачи по доработке

1. **Стейт-менеджемент (Zustand - `appStoreTypes.ts`, `uiSlice.ts`)**:
    *   Добавить в глобальный стор переменные для боковой панели:
        *   `isSidebarOpen: boolean` (состояние показа, по умолчанию `false`).
        *   `sidebarContent: ReactNode | null` - компонент или идентификатор плагина, который рендерит содержимое в данный момент (в идеале переиспользовать registry, храня `activeSidebarPluginId: string | null`).
        *   Возможно, `sidebarWidth: number` для хранения пользовательского ресайза панели (по умолчанию `350px`).
    *   Добавить экшены: `openSidebar(pluginId)`, `closeSidebar()`.

2. **Холдер в макете (Layout - `App.tsx` / `BoardView.tsx`)**:
    *   Добавить левую панель рядом с основным содержимым (доской `BoardView`).
    *   Разместить панель **слева** по умолчанию, как более привычный паттерн для чатов и инструментов редактирования.
    *   Имплементировать ручку-Splitter для легкого изменения ширины и возможности свернуть панель перетягиванием.

3. **API для плагинов (`src/plugins/registry.ts`)**:
    *   Расширить определение плагина `PluginDefinition`, добавив опциональный `sidebarComponent?: React.ComponentType`.
    *   Расширить `PluginAPI`: добавить экспортируемые методы:
        *   `openSidebar(): void` (говорит приложению открыть боковую панель этого плагина)
        *   `closeSidebar(): void`
    *   Добавить визуальные триггеры - если плагин предоставляет `sidebarComponent`, в интерфейс пользователя (например, в верхнее меню `Header` или в панель навигации слева) могут добавляться кнопки быстрого вызова.

4. **Интеграция зависимостей**:
    *   Возможно, стоит использовать простейшую реализацию resizer без тяжелых библиотек, либо использовать `<div className="resize-x" />` с нужными стилями для левой панели.
