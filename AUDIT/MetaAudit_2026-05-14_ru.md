## 🔴 КРИТИЧЕСКИЕ (P0) — Целостность данных и безопасность

### C-1: Глобальный мутабельный кэш в tree.ts

| Параметр | Значение |
|----------|----------|
| **Файл** | `src/utils/tree.ts:8-9` |
| **Статус** | ✅ ИСПРАВЛЕНО 14.05 — кэш удалён |
| **Упоминания** | MiniMax, OpenCode, DeepSeek (3/5) |
| **Суть** | Модульные переменные `_cachedNodes` и `_cachedTreeIndex` создают глобальный кэш, общий для всех потребителей. Проверка `nodes === _cachedNodes` сравнивает по ссылке. |
| **Риск** | При переключении документов кэш может отдать устаревший `TreeIndex` от предыдущего документа, если ссылка массива совпала. Утечка памяти: `_cachedTreeIndex` держит весь граф. |
| **Смягчение** | На практике Zustand всегда создаёт новый массив при `setNodes`, поэтому ссылка всегда меняется. Реальный impact — утечка памяти (holds previous doc's tree), не data corruption. |

**Рекомендация:** Заменить модульный кэш на `WeakRef` или передавать индекс как возвращаемое значение без кэширования (функция быстрая, ~1мс на 1000 узлов).

```diff
- let _cachedNodes: PuuNode[] | null = null;
- let _cachedTreeIndex: TreeIndex | null = null;
-
- export const buildTreeIndex = (nodes: PuuNode[]): TreeIndex => {
-   if (nodes === _cachedNodes && _cachedTreeIndex) {
-     return _cachedTreeIndex;
-   }
+ export const buildTreeIndex = (nodes: PuuNode[]): TreeIndex => {
```

### C-2: Race Condition при hydration в FileSystemManager

| Параметр | Значение |
|----------|----------|
| **Файл** | `src/hooks/useFileSystem.ts:365-376` |
| **Статус** | ✅ ИСПРАВЛЕНО 14.05 — abort-контроллер добавлен в createNewFile |
| **Упоминания** | MiniMax, DeepSeek, OpenCode (3/5) |
| **Суть** | `isHydratingFile` — boolean-флаг, который выставляется в `true` перед загрузкой и в `false` после. При быстром переключении файлов есть окно, где `isHydratingFile = false` установится от первого switch, пока второй ещё не завершился. |
| **Смягчение** | `switchController?.abort()` + `signal.aborted` проверки в `switchFile()` существенно снижают риск. Однако в `createNewFile()` (строки 424-436) `isHydratingFile` ставится/снимается без abort-контроллера. |

**Рекомендация:** Заменить boolean на монотонный счётчик (`hydrationVersion++`) и проверять `if (currentVersion !== hydrationVersion) return;`.

| Параметр | Значение |

### C-5: Сравнение metadata по ссылке в diffAndEmit

| Параметр | Значение |
|----------|----------|
| **Файл** | `src/store/slices/historySlice.ts:35` |
| **Статус** | ✅ ИСПРАВЛЕНО 14.05 — заменено на JSON.stringify |
| **Упоминания** | OpenCode (1/5) |
| **Суть** | `prev.metadata !== next.metadata` — сравнение по ссылке. Если metadata — объект и был пересоздан с тем же содержимым, `diffAndEmit` отправит ложный `emitNodeUpdated`. |
| **Impact** | Низкий — plugin registry пока пуст. Станет проблемой при появлении реальных плагинов. |

**Рекомендация:** Использовать `JSON.stringify` или shallow-compare для metadata, либо документировать контракт «metadata должна быть stable reference».

## 🟠 ВЫСОКИЕ (P1) — Производительность

### H-1: Undo/Redo хранит полные снимки

| Параметр | Значение |
|----------|----------|
| **Файл** | `src/store/slices/historySlice.ts:84-91` |
| **Статус** | ⚠️ АКТУАЛЬНО |
| **Упоминания** | StepFun, DeepSeek (2/5) |
| **Суть** | Каждый шаг undo сохраняет полную копию массива `nodes`. При 1000 узлов × 50 шагов = ~50K объектов в памяти. |
| **Смягчение** | `HISTORY_LIMIT = 50` ограничивает стек. Для типичного использования (~100-300 узлов) это приемлемо. |

**Рекомендация (P2):** Для масштабирования → перейти на structural sharing / diff-based history (immer patches). Но текущая реализация работоспособна до ~1000 узлов.
