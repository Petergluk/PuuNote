# Аудит Подготовительного AI-этапа (PuuNote)

**Обновлено:** 2026-04-29
**Текущая стадия:** Успешное завершение подготовки архитектуры для AI-плагинов.

## 1. Что было сделано (Подготовка к ИИ/Плагинам)

Все запланированные архитектурные шаги для адаптации редактора под ИИ были реализованы и протестированы:

### Разделение слоев (Document API)
Ранее состояния и бизнес-логика манипуляций с графом были смешаны в Zustand-сторе. Сейчас весь функционал по безопасной модификации дерева абстрагирован в `src/domain/documentApi.ts`. 
**Зачем это нужно:** AI-плагины могут писать в дерево без риска поломать структуру или React-стейт, а весь мутирующий процесс защищён Unit-тестами.

### Extractor Engine (Сбор контекста)
Создан `buildContextForLLM` (`src/domain/contextExtraction.ts`).
**Зачем это нужно:** Дерево сложно скармливать в LLM как есть. Этот модуль собирает древовидный контекст вокруг одной карточки: предки (путь к мысли), сама карточка, и все ее дочерние элементы с правильными Markdown-отступами. Полученный текст становится идеальным System Prompt.

### AI Metadata & Streaming UI
В схему `PuuNode` (через Zod Schema + интерфейсы) добавлено опциональное поле `metadata: Record<string, any>`. 
Это позволило внедрить визуализацию стриминга: флагом `metadata.isGenerating=true` для узлов теперь обрабатывается UI в виде shimmer-мерцания (loader), демонстрирующего карточки, которые переписываются или дописываются в данный момент, без блокировки интерфейса. Анимация прописана в Tailwind и `index.css`.

### Макро-Версионирование (Snapshots)
В IndexedDB/Dexie (`db.ts`) добавлена таблица `snapshots`. Механизмы создания и восстановления снимков находятся в `src/db/snapshots.ts`. Важно: при миграции Dexie `version(2)` учтены все старые таблицы во избежание потери данных.
**Зачем это нужно:** При RAG-поиске или структурировании 100+ узлов ИИ может ошибиться. Откат назад через стандартный 50-шаговый Undo не спасет. Snapshots — это глобальные резервные копии документа перед сложными экспериментами "в один клик".

### AI Job Runner
Реализована инфраструктура для отслеживания долгих задач (`Fact-Checking`, `Vector Embedding`, `Transcribing`). Добавлены `useJobStore.ts` для хранения стейта задач и класс `JobRunner` для инкапсуляции жизненного цикла задачи (`runJob`, `cancelJob`) c поддержкой коллбека `updateProgress`.

### Plugin Extension Registry
Реализован `PluginRegistry` (`src/plugins/registry.ts`), который позволяет плагинам динамически регистрироваться, встраивать UI (например, `cardActions` - кнопки поверх карточки) и подписываться на хуки (создание, удаление элементов).

## 2. Анализ кода после изменений

**Что было проверено:**
1. **Vitest тесты графа:** `npm run vitest` проходит успешно, все манипуляции со структурой дерева не ломают консистенцию. Инварианты сохранены.
2. **Миграция Базы Данных (Dexie):** Корректно прописаны индексы, предотвращена фатальная ошибка удаления старых коллекций в Dexie при `this.version(2)`. Обратная совместимость для существующих пользователей сохранена полностью.
3. **Безопасность типов:** Добавленное поле `metadata` учтено в `Zod`-валидаторах. 

**Вывод:** Внедренные изменения безопасны, не нарушают текущий user flow и готовы к интеграции внешних LLM скриптов.

## 3. RoadMap на следующий этап (Этап написания ИИ-скиллов)

Инфраструктура полностью закончена. Следующий этап — разработка и демонстрация конкретных ИИ-навыков (Skills), которые раскроют потенциал приложения. План по MVP плагинам:

1. **AI Structurer (Декомпозитор):**
   - *Бизнес-логика:* Взять хаотичный длинный текст из одного узла и разбить его на иерархию (тезис -> аргументы -> примеры) множества дочерних узлов.
   - *Технически:* Использовать `ContextExtraction`, отправить в Gemini 3.1 Pro (через JSON schema output), применить `documentApi.addNode` в батче, использовать `Snapshot` перед запуском.

2. **Fact Check (Опровергатель):**
   - *Бизнес-логика:* Выбрать утверждение на карточке. Запустить фоновый Web Search (через Google Search Tool). Прикрепить дочерние карточки с пометкой `metadata.aiType: fact` (зелёные/красные узлы с ссылками на источники) к выбранной карточке.
   - *Технически:* Использовать `JobRunner` для визуализации долгой задачи загрузки сайтов и прогресса поиска.

3. **Metaphor Injector / Tone Auto-Tuner:**
   - *Бизнес-логика:* Переписать карточку в академическом/творческом стиле.
   - *Технически:* Использовать `Stream UI` (`metadata.isGenerating`) для перерисовки контента на карточке прямо на глазах у пользователя.

4. **Интерфейс Job Runner & Plugin UI:**
   - Подключить `useJobStore` к UI (например, небольшая плашка "Задачи в фоне" в левом нижнем углу).
   - Внедрить рендеринг кнопок из `PluginRegistry` на карточку (поверх текущих иконок).
