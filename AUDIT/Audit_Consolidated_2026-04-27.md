# Сводный Аудит проекта PuuNote

**Дата:** 2026-04-27
**Основа:** Внутренний аудит (Gemini 3.1 Pro Preview) + Сторонний аудит (gpt-5-codex)

## 1. Введение и Статус Проекта
Проект **PuuNote** сделал большой шаг вперед, очистив кодовую базу от мусора, переведя стейт в Zustand и интегрировав IndexedDB. Базовая архитектура стала прозрачнее. 

Тем не менее, внешний аудит выявил ряд скрытых регрессий и гонок данных. Текущее состояние требует стабилизации, прежде чем добавлять новые продуктовые фичи.

## 2. Немедленные фиксы (Уже исправлено ✅)
Сразу после получения аудита были закрыты критичные проблемы, которые валили CI/CD:
1. **[Исправлено] Падение линтера из-за мутаций Refs:** В `App.tsx` мутация `colRefs.current.length` вынесена из фазы render в `useEffect`.
2. **[Исправлено] Ошибки `prefer-const`:** Линтер удовлетворен в `hooks/useFileSystem.ts`.
3. **[Исправлено] Уязвимости `npm audit` (High Severity):** Удален `vite-plugin-pwa` вместе с `serialize-javascript` (RCE-уязвимость).
4. **[Исправлено] Засорение dependencies:** Build-тулзы (`vite`, `@vitejs/plugin-react`) перенесены в `devDependencies`.

## 3. Подробный разбор текущих проблем (Action Items)

### А. Хранение данных, Асинхронность и Риск потери файлов
- [x] **Мгновенное удаление без Confirm:** Удаление файла сейчас происходит моментально (без `window.confirm` и без корзины). Это огромный UX риск.
- [x] **Опасный импорт и Reset "Tutorial":** Импорт `.md` файлов и кнопка `?` (Tutorial) вызывают `setNodesRaw`, что **без спроса перезаписывает** текущий активный документ, уничтожая Undo/Redo историю. Импорт должен создавать *новый* документ.
- [x] **Гонка `flushPendingSave`:** Автосохранение (`flushPendingSave`) не возвращает `Promise` и не ожидается (`await`) при создании, переключении или удалении файла. Возможна потеря последних набранных символов.
- [x] **Параллельное удаление в `deleteFile`:** Вызов `switchFile` внутри удаления происходит без `await`, сразу после этого параллельно идут `db.files.delete` и `db.documents.delete`.
- [x] **Миграция `localStorage` в IndexedDB:** Если внутри цикла миграции происходит сбой записи в `Dexie`, `catch` всё равно продолжает работу и стирает старые ключи в `localStorage`. Данные могут быть потеряны навсегда.
- [x] **Нет обработки `QuotaExceededError`:** Установка значений в `localStorage` (Preferences, last active file) не обернута в `try/catch`. 

### B. Парсинг дерева, Маркдауна и Логика графа
- [x] **Конфликт `---` маркеров:** Текущий парсер считает маркдаун "PuuNote-форматом" просто при наличии `---`. Это ломает обычный Markdown, YAML Front Matter и Horizontal rules. Маркером должен быть только `<!-- puunote-format: 1 -->`.
- [x] **Экспорт без экранирования собственных разделителей:** Если карточка содержит `---`, экспорт сломается и при повторном импорте разобьет карточку на куски.
- [x] **Дублирование Tree-Логики:** Построение индексов дерева, путей и потомков разбросано по `App.tsx`, `useAppStore`, `TimelineView` и модулю `markdownParser`. Требуется абстракция `utils/tree.ts`.
- [x] **Валидация целостности графа:** `zod` проверяет схему узлов по отдельности, но не проверяет граф целиком на зацикливания или сирот (Orphans).
- [x] **Спорный спуск `computeActivePath`:** Изначально казалось багом, но *откат показал, что пользователям нравится автоматическое выравнивание* колонок (auto-descent). Оставляем старое поведение! 

### C. UX, UI, и Производительность
- [x] **Размер бандла > 500 KB:** Тяжелые компоненты (`TimelineView`, `FullScreenModal`, `CommandPalette`, `react-markdown`) грузятся все сразу в главном чанке. Требуется `React.lazy`.
- [x] **Command Palette Ищет "В лоб":** Поиск не имеет `debounce`, проверяет только текущий файл (а не все документы из IndexedDB), и по нажатию `Cmd+K` не переключает состояние (`toggle`), а только открывает его.
- [ ] **Антипаттерн состояний (Functions in State):** Подписка UI на `canUndo/canRedo` как на функции состояния провоцирует лишние рендеры. Лучше использовать селекторы.
- [ ] **Отсутствие Виртуализации DOM:** Отрисовка 500+ узлов парализует браузер (нет Windowing-а ни для колонок, ни для Timeline).
- [ ] **Неиспользуемый `tailwind-merge`:** Утилита `cn()` существует, но игнорируется в проекте. Нужно либо внедрять, либо удалять.

### D. Инфраструктура и Среда
- [x] **Устаревший `.env.example`:** Очищен.
- [x] **Слишком громкие обещания Privacy:** Смягчено.
- [x] **Задачи SEO (title, meta-теги, картинка):** Версия удалена из `<title>`, добавлены og/twitter meta-теги, сгенерирована и подключена картинка для шаринга в соцсетях (`og-image.svg`).
- [x] **Сломанный хелпер и sample_image:** Изменено на `og-image.jpeg`, которая всегда доступна локально.
- [x] **Интерактивные чекбоксы (фича из чата):** Добавлено обновление стейта markdown при клике по атрибуту `readOnly` чекбокса в `SafeMarkdown`.
- [ ] **Частичный `i18n`:** Интернационализация применена на ~10% строк, остальное - жестко вшитый русский и английский языки.
- [ ] **Отсутствие тестов:** Отсутствие защиты от регрессий при работе с парсером, миграциями хранилища и сложными drag/drop операциями.
