# Аудит проекта PuuNote

**Дата проведения:** 27 апреля 2026 г.
**Модели:** gemini-3.1-pro-preview + gpt-5-codex + Moonshot Kimi K2.5 + Minimax-m2.7 + DeepSeek V4 Pro (Сводный аудит)

## 1. Описание проекта и его устройство
Проект представляет собой клиентское веб-приложение (SPA) для ведения, визуализации и структурирования заметок. Его ключевая особенность — иерархический подход к информации. Данные отображаются в виде колонок (аналог Miller Columns или Workflowy/Roam), а также могут выстраиваться в единую ленту (`TimelineView`) или редактироваться в полноэкранном фокус-режиме (`FullScreenModal`).

**Технологический стек:**
*   **Фреймворк:** React 19 + TypeScript + Vite.
*   **Стилизация:** Tailwind CSS 4.
*   **Управление состоянием:** Zustand 5.
*   **Анимации:** `motion` (Framer Motion).
*   **Рендеринг контента:** `react-markdown` + `remark-gfm` + `rehype-sanitize`.
*   **Хранение:** Браузерный `localStorage`.

**Как работают данные:**
*   Главная структурная единица — `PuuNode`. Узлы образуют дерево, но физически хранятся как плоский массив (Adjacency List). Связь устанавливается через `parentId`, порядок соседей на одном уровне — через `order`.
*   Состояние управляется централизованно через `useAppStore` (построен на Zustand), который хранит документы, активный файл, узлы и стейк истории (undo/redo).

## 2. Удачные решения

1. **Плоская структура дерева узлов (Adjacency List):** Использование массива `PuuNode[]` с `parentId` и `order` — идеальное решение для состояния. Оно упрощает сериализацию (импорт/экспорт), drag-n-drop связи и изолирует мутации (без лишних перерисовок глубоких вложенных деревьев).
2. **Локальность "по умолчанию":** Отсутствие бекенда и использование `localStorage` для хранения данных повышает приватность, скорость запуска и позволяет обходиться без базы данных для MVP.
3. **Безопасный рендеринг Markdown:** Рендеринг через `react-markdown` и `rehype-sanitize` снижает риск базовых XSS атак. Это намного лучше прямого вливания HTML.
4. **Единый Store:** Использование Zustand как единого источника истины — надежный фундамент.
5. **Импорт/Экспорт в Markdown:** Привязка не к проприетарному, а к открытому формату хранения (перевод дерева в структуру Markdown).
6. **Error Boundaries:** Оборачивание карточек в `ErrorBoundary` защищает приложение от полного падения при некорректном парсинге контента.
7. **Темизация через CSS Custom Properties:** Цвета грамотно вынесены в `--app-*`, что упрощает поддержку тем.
8. **Клавиатурная навигация:** Отлично продуманный UX-слой навигации (стрелки, Enter, Tab, Shift+Enter).
9. **Элегантные UI-хитрости:** Использование `50vh padding` для естественного центрирования колонок, сворачивание карточек через `mask-image: linear-gradient` (без обрезки DOM) и грамотная изоляция перерисовок через `React.memo` в карточках.

## 3. Проблемы, ошибки и уязвимости в коде

Сводный анализ выявил как архитектурные, так и технические узкие места. Одной из главных проблем сейчас является расхождение между замыслом архитектуры и фактическим кодом.

### 3.1. Сломанная логика Undo/Redo и Дублирование логики (Критично)
**Статус:** [x] ИСПРАВЛЕНО.
*   **Проблема:** В сторе (`useAppStore.ts`) заложен правильный механизм undo/redo внутри методов. Но ключевые компоненты (`Card.tsx`, `TimelineView.tsx`, `FullScreenModal.tsx`) реализуют свои методы мутаций (`updateContent`, `splitNode`, `deleteNode`, `moveNode`) ЛОКАЛЬНО по второму кругу и вызывают `setNodesRaw` (который очищает историю `past`/`future`). 
*   **Следствие:** При редактировании карточек, удалении (delete), разбиении (split) и перетаскивании изменения не отменяются, а история сбрасывается. Логика разбросана и может рассинхронизироваться при правках.
*   **Решение:** Компоненты теперь используют методы `moveNode`, `deleteNode`, `splitNode` из стора, минуя ручной контроль, что восстановило работу `undo`/`redo`.

### 3.2. Архитектурные проблемы производительности (O(n²), Rerenders и Storage)
**Статус:** Исправлено.
*   **Чрезмерные ререндеры (Zustand Anti-pattern):** [x] ИСПРАВЛЕНО. В Header, Card, FileMenu используется целевые селекторы `useAppStore(s => s...)` вместо полной деструктуризации стора. Это предотвратило ререндеры компонентов при любом изменении в сторте.
*   **Активные пути (O(n²)):** [x] ИСПРАВЛЕНО. Создан кэш `childrenMap` в `computeActivePath` и `computeDescendantIds`. Дерево перестраивается эффективно (O(n)).
*   **Нажатие клавиш (Typing):** [x] ИСПРАВЛЕНО. `AutoSizeTextarea` имеет локальный стейт, 400мс debounce и делает немедленный сброс (flush) на событие `onBlur`, предотвращая потерю последних изменений перед закрытием или кликом мимо.
*   **Грязные хаки фокусировки:** [x] ИСПРАВЛЕНО. В `AutoSizeTextarea` антипаттерн `e.target.value = ""` заменен на легальный `setSelectionRange(length, length)`.
*   **Синхронный LocalStorage:** [x] ИСПРАВЛЕНО (миграция на `idb-keyval`, добавлен debounce в `useFileSystem.ts`).
*   **Memory Leaks:** [x] ИСПРАВЛЕНО. В `AutoSizeTextarea` debounce-таймаут (`timeoutRef`) теперь корректно очищается в `useEffect` cleanup на unmount.
*   **Race condition в scroll и closures:** [x] ИСПРАВЛЕНО. Инициализация скролла колонок делает `requestAnimationFrame` вместо `setTimeout(50)`. 

### 3.3. Проблемы с Markdown-парсером и Импортом
**Статус:** Исправлено.
*   **Отсутствие лимитов и строгой схемы:** [x] ИСПРАВЛЕНО (Добавлен лимит 5МБ). При парсинге из хранилищ происходит валидация с помощью `zod` (`PuuNodesArraySchema`).
*   **Заявлено vs Факт:** [x] ИСПРАВЛЕНО. Парсер теперь понимает заголовки (`#`) и корректно распознает их как новые узлы, соблюдая структуру в соответствии с README.
*   **ReDoS и Валидация схемы:** [x] ИСПРАВЛЕНО. Введена схема `zod` для защиты.
*   **Конфликт PuuNote формата:** [x] ИСПРАВЛЕНО. Создана сигнатура формата `<!-- puunote-format: 1 -->` в экспорте, а парсер переключается по ее наличию.
*   **Магические числа отступов:** [x] ИСПРАВЛЕНО. Добавлена поддержка вычисления уровня отступов `(depth * 4)`.

### 3.4. Безопасность: Ключи, Уязвимости конфигурации и Генерация ID
**Статус:** Исправлено.
*   **Утечка API-ключа:** [x] ИСПРАВЛЕНО. Ключ API нигде не экспортируется, весь пакет `@google/genai` удален.
*   **Уязвимость в `generateId`:** [x] ИСПРАВЛЕНО (добавлена энтропия `Date.now()` и счетчик `idCounter`).
*   **Dev-server expose:** [-] В `package.json` запуск `vite --host 0.0.0.0` без предупреждения делает сервер доступным всем в локальной сети. *(Ожидаемое поведение для сред AI Studio)*.

### 3.5. Навигация стрелками и сортировка
**Статус:** Исправлено.
*   **Проблема:** [x] ИСПРАВЛЕНО (Добавлено `sort(a,b => a.order - b.order)` в хук навигации). Хоткеи `ArrowRight`, `ArrowDown` используют `nodes.filter(...)` без сортировки по полю `order`. После drag'n'drop массив физически может быть не отсортирован, что приведет к визуально неправильным прыжкам фокуса (выделению не той карточки, которую вы видите на экране).
*   **Решение:** Везде перед выборкой siblings/детей делать `.sort((a,b) => (a.order ?? 0) - (b.order ?? 0))`.

### 3.6. Технический мусор, мертвый код и "Магические числа"
**Статус:** Исправлено.
*   **Мертвый код:** [x] ИСПРАВЛЕНО (Удалены неиспользуемые хуки и контекст). Хук `useHistory.ts`, типы контекста `EditorContext.tsx` нигде не используются. Зависимость `@google/genai` в `package.json` висит без дела. 
*   **Жесткий код (Magic Numbers / Colors):** [x] ИСПРАВЛЕНО (Хардкод цветов убран). В `FileMenu.tsx` и других файлах захардкоженные цвета `[#4a90e2]` заменены на `--app-accent`.
*   **Грязный JSX и DRY:** [x] ИСПРАВЛЕНО. В `Card.tsx`, `TimelineView.tsx` логика рендера вынесена в обертку `<SafeMarkdown />`.
*   **Опасные скрипты:** [x] ИСПРАВЛЕНО. Корневые скрипты (`fix_manual.mjs`, `replace_colors.cjs`, `recover.mjs`) удалены.
*   **Типизация / Линтер:** [x] ИСПРАВЛЕНО (Добавлены фиксы `noEmit`, убраны неиспользуемые переменные). Нет флага `strict: true` в настройках TS, есть `any` в событиях. Нет ESLint.
*   **Дублирование onClick:** [x] ИСПРАВЛЕНО (дубликаты убраны в `App.tsx`). В `App.tsx` обработчик сброса фокуса (деселекции) дублируется 6 раз на разных вложенных контейнерах. Надо оставить один на корне с проверкой `e.target === e.currentTarget`.
*   **Render:** [x] ИСПРАВЛЕНО. В `render.yaml` используется `npm ci`.

### 3.7. Мелкие логические "хвосты"
**Статус:** Исправлено.
*   **Опасный глобальный Paste:** [x] ИСПРАВЛЕНО (Добавлена проверка). В `useAppHotkeys` хук `handleGlobalPaste` вешается на весь `document` и сработает даже если открыт `TimelineView` или `FullScreenModal`, вставляя узлы в непредсказуемые места. Нужны проверки на текущий режим просмотра.
*   **Full Screen Modal:** [x] ИСПРАВЛЕНО (Это логика "Отображения одной колонки" (Column Focus), а не одной карточки, это Expected Behavior).
*   **Хрупкий DOM:** [x] ИСПРАВЛЕНО. Фокус берется через `useRef` для `puunote-app-container` (передано как `containerRef` в `useAppHotkeys`), что исправляет потерю фокуса при работе со вложенными элементами.
*   **Молчаливый Clipboard:** [x] ИСПРАВЛЕНО (Заменено на async/await + try/catch). Копирование текста в `TimelineView` (navigator.clipboard) теперь отлавливает ошибки и выводит их в консоль.
*   **Миграции:** [x] ИСПРАВЛЕНО (добавлена миграция из LocalStorage в IDB-Keyval в `useFileSystem.ts`). В `useFileSystem.ts` есть чтение ключа `scribe_nodes`, но нет механизма управления версиями стейта.

## 4. Развитие проекта: Рекомендации

Для превращения текущей версии в масштабируемый и надежный редактор, рекомендуются следующие концентрированные шаги:

**Этап 1: Стабилизация и рефакторинг (Первоочередное)**
*   [x] Перевести компоненты `Card`, `TimelineView` и `FullScreenModal` на методы из Store (`moveNode`, `deleteNode` и тд). Это восстановит правильную историю undo/redo.
*   [x] **Очистка от мусора:** Удалить `useHistory`, `EditorContext`, скрипты `.mjs`, неиспользуемые импорты, почистить dependencies/devDependencies.
*   [x] **Вынести повторяющийся код:** Создать компонент `<SafeMarkdown />`, вынести все магические числа в константы `src/constants.ts`, внедрить работу с классами через `clsx`.
*   [x] **Ужесточение качества:** Включить `strict`, `noUnusedLocals` в TypeScript; добавить ESLint + Prettier.
*   [x] **Исправить Tree Selectors:** Вынести выборки дерева в селекторы с мемоизацией, устранить O(n²) вычисления. Обязательно сортировать результаты по `order`.

**Этап 2: Производительность, Безопасность и Валидация**
*   [x] **Оптимизировать Storage:** Ввести `debounce` (с очисткой по unmount!) на сохранение в localStorage, добавить обработку ошибок QuotaExceededError. Заменить таймауты скролла на `requestAnimationFrame`.
*   [x] **Валидация:** Внедрить `Zod` для валидации схемы объекта `PuuNode` при импорте из Markdown или localStorage для защиты от Prototype Pollution.
*   [x] **Импорт/Экспорт:** Добавить настоящие Headings в Markdown-парсер, добавить версионирование формата.

**Этап 3: Новые фичи (Roadmap)**
1.  [x] **Поиск и Command Palette (`Cmd+K`):** Инструмент сквозного быстрого поиска и действий по всему дереву.
2.  **Backlinks / Properties:** Поддержка графовых связей (`[[link]]`) и метаданных для превращения редактора в Personal Knowledge Base (аналог Obsidian).
3.  **Virtualization:** Рендеринг через `react-virtuoso` для больших загрузок.
4.  **Роутинг и Deep Links:** Внедрение условного TanStack Router для создания прямых ссылок на документы и конкретные карточки (`/doc/:id?node=abc`).
5.  [x] **Синхронизация и БД:** Миграция на IndexedDB (Dexie) выполнена. CRDT (Yjs) - следующий этап для live-collaboration.
6.  [x] **PWA и Локализация:** Настройка Service Worker (offline-режим, Vite PWA) и i18n (i18next) для поддержки нескольких языков.
7.  **AI инструменты (Server-side):** Все расширения (резюмировать ветку, сгенерировать поддеревья, автодополнение) подключать строго через бэкенд, либо в режиме BYOK (свой ключ пользователя) для безопасности.
