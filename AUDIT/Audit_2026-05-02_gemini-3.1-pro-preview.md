# Аудит проекта PuuNote

**Дата проведения:** 02 мая 2026 года
**Аудитор:** Gemini 3.1 Pro Preview (Комплексный анализ)

---

## 1. Описание проекта и его устройство

**PuuNote** — это специализированное Single-Page Application (SPA) для ведения заметок и структурного мышления по принципам графа/дерева (tree-based outliner). Приложение написано на современном стеке: React 19, TypeScript, Vite. 

### Как он устроен:
- **Local-first архитектура:** Основным источником истины является локальная память браузера (IndexedDB используется через удобную обертку Dexie.js + резервный SessionStorage для страховки от закрытия вкладок). На стороне сервера данные целенаправленно не хранятся, что даёт отличную приватность и офлайн-возможности.
- **Модель Данных:** Вся база документов это плоский, не вложенный массив `PuuNode[]` в глобальном хранилище `Zustand`. За счет вычисляемого `buildTreeIndex` на лету строятся отношения Parent-Child-Sibling. Это позволяет избежать проблем с древовидной сериализацией и рекурсией при чтении.
- **Интерфейс:** Реализовано два основных режима просмотра:
  1. `BoardView` (колонки и хлебные крошки, классический Miller Columns UX).
  2. `TimelineView` (плоская виртуализированная лента на базе алгоритма Depth-First Search).
- **Редактор:** Узлы редактируются с помощью классического Markdown-инпута (RAW), либо через WYSIWYG на базе `Tiptap`.

## 2. Удачные решения в проекте

1. **Плоская графовая модель данных в стейте.** Хранение дерева как плоского массива `nodes` — это отличное архитектурное решение. Оно облегчает миграции, экспорт/импорт, Undo/Redo и сохранение (flattening).
2. **Виртуализация длинных списков (`TimelineView`).** Использование библиотеки `react-virtuoso` для таймлайна гарантирует, что просмотр даже 10 000 узлов не уронит DOM (хотя `BoardView` все еще уязвим).
3. **Безопасность (Sanitize) и Zod-валидация.** Рендер Markdown фильтруется через `rehype-sanitize`, а все импортируемые данные жёстко парсятся через Zod-схему в `schema.ts`. Это защищает приложение от XSS при вставке контента из буфера.
4. **Слой `documentApi` (`src/domain/documentTree.ts`).** Вся сложная математика создания узлов, перестановки, удаления вынесена в чистые JS-функции. UI-компоненты просто вызывают функции для обновления Zustand, это делает код исключительно удобным для Unit-тестов.
5. **Снапшоты "Dirty Backup".** Сохранения перед `beforeunload` гарантируют, что внезапно закрытый браузер не заставит пользователя потерять результаты последнего часа работы.

## 3. Найденные архитектурные проблемы в коде (Узкие места)

Во время проверки текущего дерева исходников были подтверждены следующие недостатки:

1. **Гонка стейтов при загрузке (Риск потери данных) - `useFileSystem.ts`**
   Проблема `hydration` (инициализации) все еще присутствует. Между моментом считывания ID активного файла и загрузкой массива узлов из IndexedDB `useAppStore.subscribe` может ошибочно поймать момент, когда загружен только пустой `INITIAL_NODES`. Это активирует таймер, который попытается "сохранить" нулевой стейт в базу, безвозвратно затирая оригинальный старый файл.

2. **Зависание цикла (Бесконечный Loop) при удалении нод - `documentTree.ts (line 148)`**
   Функция `deleteNode` использует очередь `while (head < queue.length)`, и добавляет новые id в удаление `idsToRemove.add(curr)`. Однако в ней напрочь отсутствует проверка (visited guard/continue), был ли этот ID уже посещен. Если зловредный импорт или баг создаст циклическое древо (нода ссылается на саму себя или своего предка через `parentId`), то вкладка браузера зависнет навсегда.

3. **Огромные лаги при вводе текста (Мутация `PuuNode[]` массива)**
   Каждое нажатие на клавишу в редакторе (RAW) вызывает `documentApi.updateContent`, которое мапит **весь массив** через `.map(n => ...)`. Это классическая O(N) мутация. Если в базе будет 2-3 тысячи узлов, печать текста превратится в слайдшоу.
   *Возможное решение:* Перевести хранилище `Zustand` с плоского массива на хэшмэп `Record<string, PuuNode>`.

4. **Тихая потеря данных при ошибках валидации - `schema.ts (line 76-83)`**
   При импорте больших файлов (например, дампа из чужой программы), если попадаются дубликаты `id`, код их просто дедуплицирует записывая один поверх другого в Map (`nodeMap.set(node.id, node);`). Множество дочерних элементов лишатся своего контекста, а часть текста тупо сотрётся, хотя отчет напишет просто "was deduplicated".

5. **Глобальный кэш `buildTreeIndex` - `tree.ts`**
   Функция индекса графа глобально кэшируется (по ссылке на массив). И если внутри функции экспорта начать сортировать детей in-place (с помощью `sort` на `childrenMap`), как это делает `markdownParser.ts`, мы повредим глобальный UI-стейт. Экспортируемые данные не должны мутировать глобальный индекс.

## 4. Ошибки, недочеты и глюки UI & UX

1. **Мертвый тулбар редактора Tiptap (`WysiwygEditor.tsx`)**
   Иконки для визуального редактора выполнены в виде SVG без атрибутов `aria-label` и без текстовых подписей/тултипов (`title`). Скринридеры видят только "Button. Button.", а новички будут теряться в незнакомых иконках (особенно без hover). 
   *Глюк вставки ссылок:* при добавлении ссылки выскакивает браузерный агрессивный `window.prompt` — это допотопно, выглядит некрасиво и блокирует UI (а на смартфонах вызывает зависание виртуальных клавиатур).

2. **Непродуманная загрузочная (пустая) страница (`BoardView.tsx`)**
   Экран без данных содержит крошечную кнопку "Add Fragment". Для приложения такого уровня необходим полноценный Onboarding (Empty State Screen), где будет большой Call to Action, предложение импорта или кнопка "Открыть туториал". Сейчас юзера встречает белая стена, что сбивает с толку.

3. **Невозможность Touch-манипуляций (DnD)**
   Для работы с графом заявлены переносы (Reparenting), они работают на нативном HTML5 Drag and Drop. У этого API нет fallback'ов на сенсорных экранах или клавиатуре (например, сэндвич-меню с "Move Node Up/Down..."). Использование на iPad фактически невозможно.

4. **Выключить "Zen Mode" нельзя кнопкой "Esc"**
   Зен режим убирает шум с экрана, но для выхода в самом верху висит постоянная, мешающаяся кнопка. Отсутствие стандарта "нажми Escape чтобы вернуться" усложняет флоу.

5. **Недоделанный i18n (Хардкод строк)**
   В `App.tsx`, `JobPanel.tsx` и `SnapshotPanel.tsx` тексты вставлены напрямую в JSX ("Clear", "Loading timeline...", "Cancel job"). Обещанный RU-интерфейс из `README` спотыкается о куски английского мусора на самом важном месте.

## 5. Итоговый чеклист найденных проблем

**Критические (Угроза потери/искажения данных):**
- [ ] **Архитектура:** Устранить гонку состояний при гидратации `useFileSystem.ts`, предотвратив запись пустого массива поверх существующих заметок.
- [ ] **Data Loss:** Запретить тихую перезапись дубликатов `id` в `schema.ts:validateNodesWithReport`. Дубли должны регенерировать случайные ID и переназначать своих `children`, а не стираться.
- [ ] **UI Freeze / Security:** Добавить `if (idsToRemove.has(curr)) continue;` в алгоритм обхода `documentTree.ts:deleteNode`, чтобы ловить спам от циклических зависимостей внутри графа.

**Средние (Оптимизации производительности):**
- [ ] **Мутация O(N):** Переработать стейт Zustand с массива на объект (`Record<string, PuuNode>`) для устранения зависания при написании длинных заметок.
- [ ] **Bundle Size / Lazy Load:** WYSIWYG на основе `Tiptap` весит под мегабайт и тянет главную страницу вниз. Требуется вынести `WysiwygEditor.tsx` в `React.lazy()` чтобы он не входил в основной бандл.
- [ ] **TreeIndex In-place mutation:** В `markdownParser.ts` заменить сортировку массива с `childrenMap` на безопасную конкатенацию: `const children = [...childrenMap.get()].sort();`.

**Визуальные и UX недочеты:**
- [ ] Добавить всем иконкам Tiptap вёрстки (в BubbleMenu и тулбарах) `aria-label` и `title` для обеспечения доступности.
- [ ] Заменить вызов `window.prompt` при вставке ссылки на кастомный интерактивный popover.
- [ ] Создать богатый "Empty State" при отсутствии нод с кнопками "New Card", "Import Markdown" и "Tutorial".
- [ ] Провести i18n-проверку (поиск захардкоженных строк 'Cancel job', 'Snapshots', 'Add Fragment') и вынести их в систему переводов (`i18next`).
- [ ] Добавить в Card меню возможность "Сдвинуть вверх/сдвинуть вниз" с клавиатуры, для устройств без возможности мышного Drag'n'Drop.
- [ ] Добавить слушатель `keydown` на Escape для комфортного выхода из "Zen Mode".
