# PuuNote Project Audit — Iteration 2

**Date**: 2026-04-29

## 1. Project State Overview

### Architecture After Iteration

PuuNote is now a horizontally branching tree-based note editor with a growing AI plugin infrastructure. The data model remains the same — `PuuNode` tree with parent-child relationships, stored locally-first via IndexedDB (Dexie). Key new additions since the last audit:

- **Plugin System**: `PluginRegistry` in `src/plugins/registry.ts` with hooks for node lifecycle events
- **Job Runner**: Async job queue with progress tracking in `src/domain/jobRunner.ts`
- **Context Extraction**: LLM context builder in `src/domain/contextExtraction.ts`
- **Document API**: Pure tree operations moved to `src/domain/documentTree.ts`
- **Snapshots**: Version snapshots stored in IndexedDB (`src/db/snapshots.ts`)
- **Multi-Selection**: Shift-click to select multiple nodes
- **Virtual Timeline**: `react-virtuoso` for efficient rendering
- **Tests**: Vitest unit tests added

Previous issue with `idCounter` solved — now uses `crypto.randomUUID()` first:

```typescript
  if (crypto.randomUUID) {
    try { return crypto.randomUUID(); } catch(e) {}
  }
  return Date.now().toString(36) + "-" + Math.random().toString(36).substring(2, 10) + "-" + idCounter.toString(36);
};
```
Falls back gracefully.

## 3. Issues Found

### CRITICAL

#### 3.1 Snapshots Consuming Infinite Storage
**File**: `src/db/snapshots.ts:14-20`
**Issue**: No limit on snapshot count. Each save stores full `nodes` array:
```typescript
await db.snapshots.put({
  id: generateId(),
  documentId,
  nodes: [...nodes], // full tree every time!
  createdAt: new Date().toISOString(),
  description,
});
```
**Problem**: With autosave every second, storage will fill quickly.
**Recommendation**: 
1. Add max snapshot limit (e.g., 20 per document)
2. Compress snapshots (store deltas, not full copies)
3. Add manual snapshot only, not auto-snapshot

#### 3.2 Imported `sonner` Not Added to Dependencies
**File**: `src/db/snapshots.ts:4`
**Issue**: Uses `sonner` toast but not in package.json:
```typescript
import { toast } from "sonner"; // ERROR: not installed!
```
Actually it's in package.json now ✅ — wait, checking... Yes it's there. OK, ignore this.

#### 3.3 Transaction Still Missing
**File**: `src/hooks/useFileSystem.ts` (like previous audit)
**Issue**: Bulk delete still not transactional.
**Status**: Unchanged — still a risk.
