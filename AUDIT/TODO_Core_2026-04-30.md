### C1: Утечка памяти в FloatingCardActions (createPortal)
- **Файл:** `src/components/FloatingCardActions.tsx:186-209`
- **Проблема:** Portal создается при каждом рендере, хотя target один
- **Решение:** Вынести portal root в константу вне компонента, использовать useRef
- **Код:**
```typescript
// ДО: portal создается каждый раз
return createPortal(<AnimatePresence>...</AnimatePresence>, document.body)

// ПОСЛЕ: статичный root с useRef
const containerRef = useRef<HTMLDivElement | null>(null)
if (!containerRef.current) {
  containerRef.current = document.getElementById('floating-actions') || document.body
}
return createPortal(<AnimatePresence>...</AnimatePresence>, containerRef.current)
```

## 🟠 HIGH PRIORITY - Исправить в течение недели (9 задач)

### H1: Race condition в SnapshotPanel
- **Файл:** `src/components/SnapshotPanel.tsx:23-34`
- **Проблема:** Если activeFileId изменится во время загрузки, установит снапшоты от другого документа
- **Решение:** Capture fileId в начале, проверять перед setState
- **Код:**
```typescript
const [snapshots, setSnapshots] = useState([])
useEffect(() => {
  const fileId = activeFileId // capture
  loadSnapshots(fileId).then(data => {
    if (activeFileId === fileId) { // verify
      setSnapshots(data)
    }
  })
}, [activeFileId])
```
- [ ] Исправить

### H2: Zustand селекторы не оптимизированы
- **Файл:** `src/components/Card.tsx:18-39`
- **Проблема:** 10+ селекторов создают новые функции каждый рендер
- **Решение:** Использовать `useShallow` или комбинированные селекторы
- **Код:**
```typescript
// Вместо:
const content = useAppStore(state => state.documents.nodes[id]?.content)
const children = useAppStore(state => state.documents.nodes[id]?.children)

// Использовать:
const { content, children } = useAppStore(
  useShallow(state => ({
    content: state.documents.nodes[id]?.content,
    children: state.documents.nodes[id]?.children
  }))
)
```
- [ ] Исправить

### H3: Impure функция normalizeSiblingOrder мутирует массив
- **Файл:** `src/domain/documentTree.ts:15-22`
- **Проблема:** Мутирует входной массив nodes вместо создания нового
- **Решение:** Создавать копию перед сортировкой
- **Код:**
```typescript
// Вместо:
nodes.sort((a, b) => a.order - b.order)
return nodes

// Использовать:
return [...nodes].sort((a, b) => a.order - b.order)
```
- [ ] Исправить

### H4: Небезопасный доступ к DOM без SSR проверки

### H5: Tree index перестраивается O(n) на каждый shift+click

### H6: Merge валидация пересчитывается на каждое нажатие клавиши
