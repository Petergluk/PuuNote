# Сводный аудит и roadmap проекта PuuNote

**Обновлено:** 2026-04-28
**Модель:** Gemini 3.1 Pro

Данный документ представляет собой унифицированную выжимку из всех проведенных аудитов архитектуры и кода.

## 1. Обзор проекта и архитектуры
PuuNote — это продвинутый нелинейный (древовидный) Markdown-редактор, созданный для организации мыслей, заметок и ветвящегося контента (схема Local-first).

**Стек технологий:** 
- React 18, TypeScript, Vite
- Tailwind CSS 
- Zustand (управление состоянием, undo/redo)
- React Markdown (rehype-sanitize, remark-gfm)
- Motion (анимации), Dexie.js (IndexedDB)

**Что реализовано## 2. Успешно устранённые проблемы (по результатам всех аудитов)

### Производительность, Безопасность и Качество кода
- **Prettier & ESLint Quality Gates:** Сборка `npm run build` проходит без предупреждений и падений. Запуск автоформаттера изолирован от контента генерации (`.prettierignore` для `AUDIT/`).
- **Race Condition при `beforeunload`:** Из-за асинхронной природы IndexedDB, таймер несохранённых данных при закрытии вкладки теперь сбрасывает данные в `localStorage` («путь экстренного грязного сохранения»), который затем подхватывается и пишется в IndexedDB при следующей инициализации БД.
- **Риск Infinite Loop:** При повреждённой структуре ID в БД введён жесткий лимит итераций (`1000`) при обходе родительских и дочерних узлов дерева в `src/utils/tree.ts`.
- **Защита XSS (Markdown):** Исключён протокол `data:` из атрибута `src` в `rehype-sanitize`. Изображения ограничены только `http` и `https`. *Дополнительно:* SVG-изображения исключены из разрешенных тегов, а протокол `mailto()` удален из соображений безопасности.
- **UI Блокировки:** Убраны все нативные вызовы `alert()`. Подключён встроенный механизм `openConfirm()` из Zustand Store для информирования пользователя.
- **Глобальная чистота кода:** Magic strings (например, `<!-- puunote-format: 1 -->` и `5MB`) вынесены в `src/constants.ts`.
- [x] **Безопасность данных и автосохранение:** Внедрена строгая типизация и валидация схемы данных (`validateNodes`) с помощью Zod для отсечения некорректных узлов (например, при загрузке или "галлюцинациях" парсера). Корректно удаляются узлы с дубликатами ID.

### Визуальные исправления, UX и Функционал
- **Умное масштабирование (+ / -):** Кнопки изменения масштаба колонок теперь работают математически, пересчитывая ширину так, чтобы на экран влезало строго точное количество колонок. Учтены боковые отступы контейнера, чтобы крайние карточки не обрезались.
- **Стабилизация многоуровневых сирот (Осиротевшие колонки):** При клике на карточку, у которой нет собственных потомков, дочерние уровни от карточек-соседей больше не улетают за пределы экрана и не теряют выравнивание.
- **Синхронизация вертикального выравнивания:** Все дочерние карточки теперь выравниваются точно по центру своих родительских карточек (или ближайших к ним предков в активной ветке), чтобы ветки не "плавали".
- Перехват `Ctrl/Cmd+F` убран из `CommandPalette`, чтобы не нарушать нативный поиск по странице.
- **Кэширование Regex и хуки:** Регулярное выражение в `TimelineView` вынесено из цикла рендеринга. Добавлена корректная работа React-хуков (отсутствие warning'ов exhaustive-deps и conditional hooks в `FullScreenModal`).
- **Унификация и DRY (Don't Repeat Yourself):** Дублирующаяся логика переключения Markdown-чекбоксов (`handleToggleCheckbox`) абстрагирована в кастомный хук `useToggleCheckbox`.
- **Гонка в AutoSizeTextarea:** Исправлен Race Condition (разрыв между React state debounce и `onBlur`) — теперь введен промежуточный `pendingChangeRef`, гарантирующий отправку последних изменений.
- **Интеллектуальная Вставка (Intelligent Paste):** Хук `useAppHotkeys` теперь поддерживает как дефолтное разделение через `---`, так и вставку из нативного буфера форматов `PuuNote` (`<!-- puunote-node -->`), что ликвидирует баги при копировании кусков дерева.
- [x] **Глобальный поиск (CommandPalette):** Реализован быстрый нечеткий поиск по всем документам с помощью `Fuse.js`.
- [x] **Уведомления и ошибки:** Внедрена библиотека всплывающих уведомлений `Sonner` (сценарии ошибок импорта, нехватки квоты LocalStorage и пр.).
- [x] **Оптимизация бандла (Code Splitting):** Настроены `manualChunks` в Vite, библиотека разбила огромные вендоры на отдельные подгружаемые модули (markdown, ui-icons).
- [x] **Защита от случайных удалений:** Добавлено безопасное подтверждение через `ConfirmDialog` при попытке удалить карточку с большим количеством дочерних элементов (Soft Delete confirmation).

## 3. Дорожная карта (Roadmap) / Разделение по приоритетам

Ниже перечислены задачи, которые **еще предстоит выполнить**. 

### 3.1. Архитектурные блокеры (Необходимо для перехода к AI/API)
Эти задачи обязательны перед тем, как давать ИИ-агентам доступ к графу документа, так как они защищают целостность данных от масштабных автоматических мутаций.

- [x] **Разделение слоёв (Сепарация Store):** Сейчас `useAppStore` смешивает UI-состояние, мутации дерева, работу с файлами, экспорт и undo/redo. Необходимо выделить чистые операции с документами в отдельный слой (API-фасад), чтобы ИИ-плагины не сломали React-состояние и историю изменений (Undo/Redo).
- [x] **Транзакционность IndexedDB и генерация ID:** При массовых автоматических вставках от ИИ (генерация поддеревьев на сотни карточек) потребуется оборачивать операции в `db.transaction` (на данный момент добавлено только на удаление файла), чтобы исключить "битые" промежуточные состояния в БД при сбоях. Генератор `generateId()` должен быть на 100% устойчив к коллизиям в цикле.
- [x] **Автоматические тесты:** Перед разработкой AI-манипуляций с графом критически необходимо внедрить `Vitest` для покрытия операций (`tree.ts`). Если ИИ ошибется в логике добавления узлов (например, закольцует графы), это легко отловить Unit-тестами графа, защитив пользовательские данные.

### 3.2. Улучшения UX и Производительности (Nice to have)
Эти задачи не блокируют внедрение ИИ, но сделают работу конечного пользователя в будущем комфортнее.

- [x] **Виртуализация доски (DOM Virtualization):** Внедрена через `react-virtuoso` на уровне `TimelineView`, который страдал от наибольшей DOM-перегрузки при рендеринге плоского дерева.

## 4. Концепция развития: Плагинная система и интеграция ИИ (AI Features API)

Следующим стратегическим этапом развития проекта является превращение PuuNote из универсального редактора в **платформу для создания узкоспециализированных инструментов** (например, для сценаристов подкастов, писателей книг, научных работников или психологов). 
Это будет реализовано через систему подключаемых плагинов и выделенный API для ИИ-агентов.

### Что нужно подготовить архитектурно:

1. [x] **Plugin Manager (Менеджер расширений):**
   - Механизм динамической загрузки и выгрузки плагинов.
   - *Реестр хуков (Hooks Registry):* Добавлен базовый `PluginRegistry` (`src/plugins/registry.ts`).

2. [x] **Context Extraction Layer (Слой извлечения контекста для LLM):**
   - Добавлен модуль `buildContextForLLM` (`src/domain/contextExtraction.ts`) для сериализации контекста ветки в текст.

3. [x] **Data Isolation (Изоляция состояния плагинов):**
   - В модели узла дерева (`PuuNode`) добавлено поле `metadata?: Record<string, any>` для сохранения данных от плагинов (ИИ, фактов и т.д). Validated in Zod.

4. [x] **Инфраструктура для потоковой генерации (Streaming UI):**
   - Флаг `isGenerating` в `metadata` карточки.
   - CSS-анимация (shimmer / loader), чтобы визуализировать процесс стриминга.

5. [x] **Версионирование и Snapshots (Контроль версий откатов):**
   - Добавлена таблица `snapshots` в Dexie и утилиты макро-версионирования (`src/db/snapshots.ts`).
   - ИИ-операции могут создавать бекап текущего состояния для надежного отката (Undo из 50 мелких шагов недостаточно для макро-мутаций).

6. [x] **Менеджер фоновых задач (AI Job Runner):**
   - Добавлен `JobRunner` (`src/domain/jobRunner.ts`) и Zustand store (`useJobStore.ts`) для управления долгоиграющими задачами (RAG, транскрипция).

7. [x] **Семантический поиск и Meta-карты (RAG / Vector DB):**
   - Схема данных расширена `metadata` на уровне корня `PuuDocument` для хранения метаданных всего графа (embedding-хеши, теги).

### Первоочередные прототипы плагинов (MVP):
Вместо преждевременного создания громоздких «бандлов» (сборок), разработка должна фокусироваться на создании, тестировании и отладке единичных локальных инструментов, которые в будущем можно будет переиспользовать:
- **AI-Structurer (Утилита структурирования):** Плагин, который берет хаотичный текст из одного большого узла и разбивает его на логическое дерево дочерних карточек.
- **Fact-Checker (Инструмент валидации):** Модуль, который берет конкретное утверждение, инициирует веб-поиск (tool calling) и прикрепляет дочерние узлы с подтверждающими/опровергающими ссылками.
- **Metaphor & Tone Editor (Стилист):** Утилита переписывания карточки для изменения тональности (например, "сделай академично" или "подбери метафору"). 
- **Outline Generator (Макро-планировщик):** Плагин для генерации рыбы/сценария по корневой теме, после успешного тестирования которого можно будет собирать уже специализированные рабочие пространства (для подкастеров, сценаристов, ученых путем легкой кастомизации и добавления узкоспециализированных промптов).
