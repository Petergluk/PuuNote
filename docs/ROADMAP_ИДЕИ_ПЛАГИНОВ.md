# Стратегия развития и идеи плагинов (Сводка из аудитов)

Анализ проведенных аудитов архитектуры PuuNote показал сильные стороны проекта (надежность Zustand, IndexedDB хранилище, горизонтально-древовидная модель) и выявил наиболее перспективные направления будущего роста. 

---

## 🚀 Глобальный Roadmap (Развитие проекта)

Аудиторы сходятся во мнении, что проект является мощным фундаментом, но для выхода на уровень стабильного продукта и формирования вокруг него комьюнити требуется закрыть 4 основных вектора:

1. **Безопасность и Изоляция плагинов (Sandboxing):**
   - На данный момент плагины запускаются в главном потоке и имеют 100% административный доступ к данным (XSS уязвимость).
   - Внедрение `iframe sandbox` или WebWorkers для запуска недоверенных плагинов интеграторов.
   - Механизм разрешений (Permission System), где плагины запрашивают доступы (чтение/запись документов).

2. **Синхронизация и Кросс-платформенность:**
   - Мост к облачным хранилищам (Google Drive, GitHub Gist API) для легковесных бекапов.
   - Offline-First синхронизация с удаленным сервером с разрешением конфликтов.
   - Улучшение поддержки PWA для автономной работы.

3. **Маркетплейс и Экосистема (Plugin Marketplace):**
   - Установка сторонних плагинов "в 1 клик", минуя хардкод в исходниках.
   - Версионирование плагинов, поддержка HMR для разработки плагинов без перезагрузки страницы.
   - Вынос системы из песочницы (`PuuNote_Sandbox`) в общий унифицированный registry.

4. **Инфраструктура API ключей:**
   - Перевод запросов Gemini AI на единый защищенный серверный прокси, чтобы скрыть ключи пользователей из локального `localStorage`.

## 🧩 Рейтинг идей для плагинов

Все идеи были собраны, очищены от дубликатов и отсортированы по частоте упоминания и ценности для аудитории (Product-Market Fit).

### 🔥 Топ 1-3: Самые популярные и ожидаемые

1. **Kanban Board View (Task-Kanban Sync)**
   - *Суть:* Трансформация выбранной ветки карточек в Kanban-доску. Горизонтальные карточки становятся колонками, дочерние — задачами. Drag-and-drop перетаскивает карточки по статусам.
   - *Рейтинг:* Упоминается всеми. Превращает PuuNote в инструмент проектного менеджмента.

2. **Web Clipper / Content Parser**
   - *Суть:* Браузерное расширение для простого сохранения веб-статей прямо в ветки PuuNote. Дерево идеально подходит для структурированного чтения и разбивки длинных лонгридов.

3. **AI Smart Summarizer & Auto-Tag (Авто-структурирование)**
   - *Суть:* Используя уже встроенный Gemini, плагин анализирует ветку (или всю левую колонку), пишет сумму/резюме и проставляет автотеги. Идеально для работы с хаосом записей.

### 🌟 Топ 4-7: Визуализация и Обучение

4. **Diagram Generator (Mermaid.js / Mind-Map Export)**
   - *Суть:* Конвертация Markdown ` ```mermaid ` блоков в "живые" SVG прямо в карточках. А также возможность экспортировать горизонтальное дерево в стандартный XMind / FreeMind формат (что концептуально невероятно близко).

5. **Spaced Repetition / Flashcards (Мост с Anki)**
   - *Суть:* Древовидная структура — это идеальный набор "Вопрос -> Ответ (ветка)". Генерация flashcards для повторения конспектов прямо в интерфейсе или экспорт в формат Anki.

6. **Voice-to-Text 2.0 (Умная транскрипция)**
   - *Суть:* Развитие встроенного Voice Fixer — транскрибация длинных лекций в реальном времени с автоматическим разбиением на ветви по смыслу.

7. **Encryption Plugin (Зашифрованные ветки)**
   - *Суть:* End-to-end шифрование конкретных карточек, паролей или личных дневников. В IndexedDB попадает только шифротекст. Главный selling-point для Privacy-ориентированных пользователей.

### 🛠 Топ 8-10: Узкоспециализированные (Для гиков и ученых)

8. **Citation & Reference Manager**
   - *Суть:* Академический плагин. Распознает DOI / ISBN, скачивает BibTeX информацию через CrossRef и вставляет её в футер карточек по клику. Переключатель стилей APA, MLA, Chicago.

9. **Daily Journal (Авто-Даты и Шаблоны)**
   - *Суть:* Автоматическое создание карточек "Сегодня" по шаблону (Morning pages, Daily log). Запуск рутинных чеклистов.

10. **Code Runner (Выполнение кода)**
    - *Суть:* Для IT. Выполнение кусков кода (Python, JS) в WebWorker контейнере или удаленной песочнице с выводом результата под карточкой в качестве дочернего узла.
